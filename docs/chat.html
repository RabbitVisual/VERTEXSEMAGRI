Módulo Chat

Chat institucional completo para suporte ao público e comunicação interna. Sistema de atendimento em tempo real com widget público e painel administrativo.

Sobre o Módulo

O módulo Chat oferece um sistema completo de chat institucional para suporte ao público e comunicação interna. O sistema permite que visitantes iniciem conversas através de um widget público, enquanto atendentes gerenciam múltiplas sessões através de um painel administrativo.

O módulo inclui controle de horários de funcionamento, sistema de fila de espera, atribuição de atendentes, contadores de mensagens não lidas e histórico completo de conversas.

Categoria

Comunicação e Suporte

Funcionalidades Principais

Recursos completos para chat institucional e atendimento ao público

Chat Público

Widget público para visitantes iniciarem conversas, com coleta de informações básicas (nome, email, telefone) e identificação por IP.

Painel Administrativo

Painel completo para atendentes gerenciarem múltiplas sessões, visualizar fila de espera, atribuir conversas e acompanhar histórico.

Horários de Funcionamento

Configuração de horários de funcionamento por dia da semana, com verificação automática de disponibilidade e cálculo de próxima disponibilidade.

Sistema de Fila

Sistema de fila de espera para visitantes quando não há atendentes disponíveis, com status de aguardando, em atendimento, encerrada e transferida.

Contadores de Não Lidas

Contadores separados de mensagens não lidas para atendentes e visitantes, permitindo notificações precisas e controle de leitura.

Atribuição de Atendentes

Sistema de atribuição de conversas a atendentes específicos, permitindo distribuição de carga de trabalho e responsabilização.

Anexos e Arquivos

Suporte a envio de anexos (imagens e arquivos) nas mensagens, com armazenamento seguro e controle de tipos de mensagem.

Rastreamento de Atividade

Rastreamento de última atividade de cada sessão, permitindo identificar conversas inativas e otimizar atendimento.

Chat Interno

Suporte a conversas internas entre usuários do sistema, permitindo comunicação entre equipes e departamentos.

Estrutura de Dados

O módulo utiliza três tabelas principais: chat_sessions, chat_messages e chat_configs.

Tabela: chat_sessions

Campo Tipo Descrição
id bigint Identificador único
session_id string (unique) ID único da sessão de chat
type enum Tipo: 'public' (público) ou 'internal' (interno)
visitor_name string (nullable) Nome do visitante
visitor_email string (nullable) Email do visitante
visitor_phone string (nullable) Telefone do visitante
visitor_ip string (nullable) IP do visitante
user_id foreignId (nullable) Usuário que iniciou a sessão (para chat interno)
assigned_to foreignId (nullable) Atendente atribuído à sessão
status enum Status: 'waiting', 'active', 'closed', 'transferred'
unread_count_user integer Contador de mensagens não lidas pelo atendente
unread_count_visitor integer Contador de mensagens não lidas pelo visitante
last_activity_at timestamp (nullable) Data/hora da última atividade na sessão
closed_at timestamp (nullable) Data/hora de encerramento da sessão
notes text (nullable) Notas internas sobre a sessão

Tabela: chat_messages

Campo Tipo Descrição
id bigint Identificador único
chat_session_id foreignId Sessão de chat relacionada
user_id foreignId (nullable) Usuário que enviou a mensagem (se aplicável)
sender_type enum Tipo de remetente: 'user', 'visitor' ou 'system'
message text Conteúdo da mensagem
message_type enum Tipo: 'text', 'image', 'file' ou 'system'
attachment_path string (nullable) Caminho do anexo (se aplicável)
attachment_name string (nullable) Nome original do anexo
is_read boolean Indica se a mensagem foi lida
read_at timestamp (nullable) Data/hora de leitura da mensagem

Tabela: chat_configs

Campo Tipo Descrição
id bigint Identificador único
key string Chave da configuração (ex: 'chat_enabled', 'opening_hours')
value text Valor da configuração (pode ser JSON para estruturas complexas)
description text (nullable) Descrição da configuração

Relacionamentos

Os modelos do módulo Chat mantêm relacionamentos simples e diretos, focados em sessões e mensagens.

ChatSession

  • user(): belongsTo - Usuário que iniciou a sessão (chat interno)
  • assignedTo(): belongsTo - Atendente atribuído à sessão
  • messages(): hasMany - Todas as mensagens da sessão (ordenadas por data)
  • lastMessage(): hasOne - Última mensagem da sessão

ChatMessage

  • session(): belongsTo - Sessão de chat relacionada
  • user(): belongsTo - Usuário que enviou a mensagem (se aplicável)

Métodos e Scopes

Os modelos do módulo Chat possuem métodos auxiliares e scopes que facilitam consultas, controle de leitura e gerenciamento de sessões.

Scopes (ChatSession)

  • scopeActive() - Filtra sessões ativas (waiting ou active)
  • scopeWaiting() - Filtra sessões aguardando atendimento
  • scopeAssignedTo() - Filtra sessões atribuídas a um usuário
  • scopePublic() - Filtra sessões públicas
  • scopeInternal() - Filtra sessões internas

Accessors

  • status_texto - Status formatado em texto
  • is_active - Indica se a sessão está ativa
  • has_unread_messages - Indica se há mensagens não lidas
  • sender_name - Nome do remetente (ChatMessage)
  • formatted_message - Mensagem formatada com quebras de linha (ChatMessage)

Métodos Auxiliares

  • markAsRead() - Marca mensagens como lidas (ChatSession e ChatMessage)
  • incrementUnread() - Incrementa contador de não lidas (ChatSession)
  • assignTo() - Atribui sessão a um atendente (ChatSession)
  • close() - Encerra uma sessão (ChatSession)
  • isEnabled() - Verifica se chat está habilitado (ChatConfig)
  • isPublicEnabled() - Verifica se chat público está habilitado (ChatConfig)
  • isAvailableNow() - Verifica se chat está disponível no momento (ChatConfig)
  • getNextAvailability() - Retorna próxima disponibilidade (ChatConfig)
