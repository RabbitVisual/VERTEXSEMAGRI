Módulo Central de Atendimento

Módulo Demandas

Sistema central de gestão de demandas da população, solicitações e atendimentos do município. Integração completa com todos os módulos de infraestrutura e geração automática de ordens de serviço.

Sobre o Módulo

O módulo Demandas é o coração do sistema VERTEXSEMAGRI, funcionando como o ponto central de entrada para todas as solicitações da população. Ele gerencia o ciclo completo de atendimento, desde o registro inicial até a conclusão, integrando-se com todos os módulos de infraestrutura e operacionais.

Este módulo permite o cadastro de solicitações relacionadas a água, iluminação, estradas e poços, com controle de prioridades, status e geração automática de ordens de serviço quando necessário.

Categoria

Funcionalidades Principais

Recursos completos para gestão centralizada de demandas e atendimentos

Cadastro de Solicitações

Registro completo de solicitações da população com informações do solicitante, localização, tipo de demanda e prioridade.

Tipos de Demanda

Suporte a 4 tipos principais: água, iluminação (luz), estradas e poços, cada um com características específicas.

Controle de Prioridades

Sistema de priorização com 4 níveis: baixa, média, alta e urgente, permitindo organização eficiente do atendimento.

Status de Acompanhamento

Controle completo do ciclo de vida: aberta, em andamento, concluída e cancelada, com histórico de alterações.

Geração Automática de OS

Criação automática de ordens de serviço a partir de demandas, integrando com módulos de Equipes e Materiais.

Anexo de Fotos

Suporte para anexo de múltiplas fotos por demanda, facilitando a documentação visual dos problemas relatados.

Integração com CadÚnico

Vinculação opcional com pessoas do Cadastro Único, permitindo rastreamento completo do histórico de atendimentos.

Consulta Pública

Sistema de consulta pública por código, permitindo que cidadãos acompanhem o status de suas demandas (LGPD compliant).

Notificações Automáticas

Sistema de notificações integrado que alerta sobre mudanças de status, criação de ordens de serviço e conclusões.

Tipos de Demanda

O módulo suporta 4 tipos principais de demandas, cada um integrado com seu respectivo módulo de infraestrutura.

Água

Demandas relacionadas a problemas com abastecimento de água, carro pipa, falta de água, vazamentos, etc.

Integração: Módulo Água (redes e pontos de distribuição)

Iluminação (Luz)

Demandas relacionadas a postes quebrados, luz queimada, relé com problema, braço de poste danificado, etc.

Integração: Módulo Iluminação (pontos de luz)

Estrada

Demandas relacionadas a buracos, estrada intransitável, ponte danificada, necessidade de manutenção, etc.

Integração: Módulo Estradas (trechos de estrada)

Poço

Demandas diretamente relacionadas a poços artesianos: bomba queimada, poço sem água, problemas elétricos, etc.

Integração: Módulo Poços (com campo poco_id para vinculação direta)

Estrutura de Dados

A tabela demandas armazena todas as informações necessárias para o gerenciamento completo do ciclo de atendimento.

Campo Tipo Descrição
id bigint Identificador único
codigo string (unique) Código único gerado automaticamente para consulta pública
solicitante_nome string Nome completo do solicitante
solicitante_telefone string (nullable) Telefone de contato do solicitante
solicitante_email string (nullable) Email de contato do solicitante
localidade_id foreignId Localidade onde a demanda foi registrada (obrigatório)
pessoa_id foreignId (nullable) Pessoa do CadÚnico vinculada (opcional)
poco_id foreignId (nullable) Poço relacionado (quando tipo é 'poco')
tipo enum agua, luz, estrada, poco
prioridade enum baixa, media, alta, urgente (padrão: media)
motivo string Motivo resumido da demanda
descricao text (nullable) Descrição detalhada do problema
status enum aberta, em_andamento, concluida, cancelada (padrão: aberta)
fotos json (nullable) Array de URLs de fotos anexadas
user_id foreignId (nullable) Usuário que registrou a demanda
data_abertura timestamp Data e hora de abertura (automático)
data_conclusao timestamp (nullable) Data e hora de conclusão
observacoes text (nullable) Observações e notas adicionais

Relacionamentos

O modelo Demanda mantém relacionamentos integrados com múltiplos módulos do sistema, criando uma rede completa de informações.

Demanda

  • localidade(): belongsTo - Demanda pertence a uma localidade (obrigatório)
  • pessoa(): belongsTo - Pessoa do CadÚnico vinculada (opcional)
  • usuario(): belongsTo - Usuário que registrou a demanda
  • ordemServico(): hasOne - Ordem de serviço gerada a partir desta demanda
  • poco(): belongsTo - Poço relacionado (quando tipo é 'poco')

Métodos e Accessors

O modelo Demanda possui diversos métodos auxiliares e accessors que facilitam o trabalho com os dados.

Accessors

  • status_texto - Retorna status formatado
  • status_cor - Retorna cor do status
  • prioridade_texto - Retorna prioridade formatada
  • prioridade_cor - Retorna cor da prioridade
  • tipo_texto - Retorna tipo formatado

Scopes

  • scopeAbertas() - Filtra demandas abertas
  • scopeEmAndamento() - Filtra em andamento
  • scopeConcluidas() - Filtra concluídas
  • scopeUrgentes() - Filtra urgentes
  • scopePorTipo($tipo) - Filtra por tipo

Métodos Auxiliares

  • podeCriarOS() - Verifica se pode criar ordem de serviço
  • temOS() - Verifica se já tem ordem de serviço
  • podeConcluir() - Verifica se pode concluir
  • podeCancelar() - Verifica se pode cancelar
  • diasAberta() - Calcula dias desde abertura

Métodos Públicos (LGPD)

  • toPublicArray() - Retorna dados públicos (sem informações sensíveis)
  • podeSerConsultadaPublicamente() - Verifica se pode ser consultada
  • getOrdemServicoPublica() - Retorna dados da OS para consulta pública
