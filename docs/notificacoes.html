Módulo de Comunicação

Módulo Notificações

Sistema completo de notificações em tempo real, alertas e comunicações internas. Integração com todos os módulos e suporte a múltiplos tipos de notificações.

Sobre o Módulo

O módulo Notificações é um sistema completo de notificações em tempo real integrado ao VERTEXSEMAGRI. Ele permite enviar notificações para usuários específicos, roles ou todos os usuários do sistema, com suporte a diferentes tipos de notificações e integração com email.

O módulo oferece sistema de polling automático (30 segundos), badge de contador de notificações não lidas, interface administrativa completa e API RESTful para integração com outros módulos.

Versão

1.0.0

Categoria

Comunicação e Sistema

Funcionalidades Principais

Recursos completos para sistema de notificações em tempo real

Notificações em Tempo Real

Sistema de polling automático (30 segundos) para atualização em tempo real de notificações, sem necessidade de recarregar a página.

Múltiplos Tipos

Suporte a diferentes tipos de notificações: info, success, warning, error, alert e system, cada um com ícone e cor específica.

Destinatários Flexíveis

Envio para usuário específico, role específica ou todos os usuários do sistema, permitindo comunicação direcionada ou global.

Integração com Módulos

Fácil integração com outros módulos através do trait SendsNotifications e NotificacaoService, permitindo notificações automáticas de eventos.

Integração com Email

Envio automático de emails para notificações importantes (configurável), permitindo comunicação por múltiplos canais.

API RESTful

Endpoints completos para integração via API: listar notificações, contar não lidas, marcar como lidas e gerenciar notificações.

Interface Administrativa

Interface completa para gerenciamento de notificações: criar, visualizar, marcar como lidas e deletar notificações.

Badge de Contador

Contador de notificações não lidas em tempo real, exibido no badge do menu, atualizado automaticamente a cada 30 segundos.

Marcação de Lidas

Marcar notificações individualmente ou todas como lidas, com rastreamento de data/hora de leitura.

Estrutura de Dados

O módulo utiliza a tabela notifications para armazenar todas as notificações do sistema.

Tabela: notifications

Campo Tipo Descrição
id bigint Identificador único
type string(255) Tipo: 'info', 'success', 'warning', 'error', 'alert' ou 'system'
title string(255) Título da notificação
message text Mensagem da notificação
user_id bigint (nullable) ID do usuário destinatário (null para notificações globais)
role string(255) (nullable) Role destinatária (null para notificações individuais)
module_source string(255) (nullable) Módulo que originou a notificação (para rastreabilidade)
entity_type string(255) (nullable) Tipo da entidade relacionada (ex: 'Demanda', 'OrdemServico')
entity_id bigint (nullable) ID da entidade relacionada
is_read boolean Indica se a notificação foi lida (padrão: false)
read_at timestamp (nullable) Data/hora em que a notificação foi lida
action_url string(255) (nullable) URL de ação relacionada à notificação
data json (nullable) Dados adicionais em formato JSON

Relacionamentos

O modelo Notificacao mantém relacionamento com o modelo User e pode ser vinculado a qualquer entidade do sistema através de entity_type e entity_id.

Notificacao

  • user(): belongsTo - Usuário destinatário da notificação

Métodos e Scopes

O modelo Notificacao e o serviço NotificacaoService possuem métodos auxiliares e scopes que facilitam consultas e envio de notificações.

Scopes (Notificacao)

  • scopeUnread() - Filtra notificações não lidas
  • scopeForUser() - Filtra notificações do usuário ou globais
  • scopeForRole() - Filtra notificações por role
  • scopeForModule() - Filtra notificações por módulo

Accessors (Notificacao)

  • type_color - Cor do badge baseada no tipo
  • type_icon - Ícone baseado no tipo
  • type_texto - Texto do tipo em português

Métodos (NotificacaoService)

  • sendToUser() - Envia notificação para usuário específico
  • sendToRole() - Envia notificação para todos os usuários de uma role
  • sendToAll() - Envia notificação para todos os usuários
  • markAsRead() - Marca notificação como lida
  • markAllAsRead() - Marca todas as notificações do usuário como lidas
