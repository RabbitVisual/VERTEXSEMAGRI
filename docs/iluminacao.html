Módulo Iluminação

Gestão completa de iluminação pública, postes, lâmpadas e infraestrutura de energia. Controle de status, manutenção preventiva e integração com demandas e ordens de serviço.

Sobre o Módulo

O módulo Iluminação gerencia o cadastro completo de pontos de luz públicos do município. O sistema permite cadastrar postes, lâmpadas, controlar status de funcionamento, realizar manutenção preventiva e integrar-se com o sistema de demandas e ordens de serviço.

O módulo integra-se com o módulo Localidades para geolocalização, com Demandas para rastreamento de solicitações de iluminação, e com Ordens de Serviço para gerenciamento de manutenções.

Categoria

Infraestrutura de Energia

Funcionalidades Principais

Recursos completos para gestão de iluminação pública e infraestrutura de energia

Cadastro Completo

Cadastro completo de pontos de luz com informações técnicas: tipo de lâmpada, potência, tipo de poste, altura, tipo de fiação, datas de instalação e manutenção.

Geolocalização

Geolocalização completa com latitude e longitude, permitindo visualização em mapas e integração com sistemas de navegação e localização.

Controle de Status

Controle de status: funcionando, com defeito ou desligado, permitindo monitoramento em tempo real do estado da iluminação pública.

Manutenção Preventiva

Controle de datas de instalação e última manutenção, permitindo planejamento de manutenção preventiva e histórico de intervenções.

Integração com Demandas

Integração com o módulo Demandas para rastreamento de solicitações de iluminação na mesma localidade, permitindo histórico completo de demandas relacionadas.

Integração com Ordens

Rastreamento de ordens de serviço relacionadas através das demandas, permitindo histórico completo de manutenções e intervenções realizadas.

Estatísticas e Relatórios

Estatísticas completas: total de demandas, ordens de serviço, status de funcionamento e exportação de dados em múltiplos formatos (Excel, PDF, CSV).

Vinculação com Localidades

Vinculação obrigatória com localidades, permitindo organização geográfica e consulta por região do município.

Estrutura de Dados

O módulo utiliza a tabela pontos_luz para armazenar informações sobre pontos de iluminação pública.

Tabela: pontos_luz

Campo Tipo Descrição
id bigint Identificador único
codigo string (unique) Código único gerado automaticamente (ex: PL-LED-001)
localidade_id foreignId ID da localidade onde o ponto está localizado
endereco string(255) Endereço do ponto de luz
latitude decimal(10,8) (nullable) Latitude para geolocalização
longitude decimal(11,8) (nullable) Longitude para geolocalização
nome_mapa string(255) (nullable) Nome para identificação no mapa
tipo_lampada string(100) (nullable) Tipo de lâmpada (ex: LED, vapor_sodio, fluorescente)
potencia integer (nullable) Potência em watts
tipo_poste string(100) (nullable) Tipo de poste (ex: concreto, metalico, madeira)
altura_poste decimal(5,2) (nullable) Altura do poste em metros
tipo_fiacao string(100) (nullable) Tipo de fiação
data_instalacao date (nullable) Data de instalação
ultima_manutencao date (nullable) Data da última manutenção
status enum Status: 'funcionando', 'com_defeito' ou 'desligado' (padrão: 'funcionando')
observacoes text (nullable) Observações adicionais sobre o ponto de luz

Relacionamentos

O modelo PontoLuz mantém relacionamentos integrados com múltiplos módulos do sistema, permitindo rastreamento completo de demandas e ordens de serviço relacionadas.

PontoLuz

  • localidade(): belongsTo - Localidade onde o ponto está localizado
  • demandas(): hasMany - Demandas de iluminação na mesma localidade (tipo: 'luz')
  • ordensServico(): hasManyThrough - Ordens de serviço relacionadas através das demandas de iluminação

Métodos e Accessors

O modelo PontoLuz possui accessors que facilitam consultas e gerenciamento de estatísticas relacionadas.

Accessors

  • estatisticas - Retorna estatísticas completas do ponto de luz:
    • • total_demandas: Total de demandas relacionadas
    • • demandas_abertas: Demandas com status 'aberta'
    • • demandas_em_andamento: Demandas com status 'em_andamento'
    • • demandas_concluidas: Demandas com status 'concluida'
    • • total_ordens: Total de ordens de serviço relacionadas
    • • ordens_pendentes: Ordens com status 'pendente'
    • • ordens_em_execucao: Ordens com status 'em_execucao'
    • • ordens_concluidas: Ordens com status 'concluida'
